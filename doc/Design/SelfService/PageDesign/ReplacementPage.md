## 四、功能描述

### 4.1 页面用途

ReplacementPage 用于用户自助补办丢失或损坏的卡片。流程包括确认补卡、缴纳工本费（如需要）、制卡、领取新卡。

### 4.2 核心功能

1. **信息展示**
   - 展示用户基本信息（嵌入 Staff/Student 模块）
   - 显示原卡信息（如有）

2. **补卡确认**
   - 显示补卡须知
   - 确认补卡操作
   - 提示相关费用

3. **费用处理**
   - 计算应缴金额
   - 支持余额直接扣款
   - 或跳转支付流程

4. **制卡流程**
   - 发送制卡指令
   - 展示制卡进度
   - 检测出卡状态

5. **完成确认**
   - 显示新卡信息
   - 提示领取卡片

### 4.3 操作流程

```
显示用户信息
    ↓
显示补卡须知
    ↓
用户确认补卡
    ↓
判断费用情况
    ↓
┌──────────┴──────────┐
│                     │
余额充足             余额不足
    │                     │
直接扣款              跳转支付
    │                     │
└──────────┬──────────┘
           ↓
      开始制卡
           ↓
      制卡完成
           ↓
      提示取卡
           ↓
      检测到取卡
           ↓
      完成补卡
```

## 五、交互描述

### 5.1 状态转换

```
状态 1: 补卡确认
    ↓ 点击"确认补卡"
状态 2: 费用确认（如需要）
    ↓ 确认支付
状态 3: 制卡中
    ↓ 制卡完成
状态 4: 出卡提示
    ↓ 检测到取卡
状态 5: 完成
    ↓ 跳转
CompletionPage
```

### 5.2 详细交互

**1. 页面加载**:
- 从 Session 获取用户信息
- 嵌入 Staff/Student 信息模块
- 查询原卡状态和余额
- 显示补卡须知

**2. 确认补卡**:
- 用户点击"确认补卡"
- 检查用户是否已有在制卡片
- 计算应缴费用
- 判断余额是否充足

**3. 费用处理**:
- **余额充足**: 显示确认对话框，直接扣款
- **余额不足**: 提示需要充值，提供支付选项
- 支持跳转 RechargePage 充值后再补卡

**4. 制卡过程**:
- 发送制卡指令到设备
- 显示制卡动画
- 等待制卡完成
- 检测出卡状态

**5. 完成流程**:
- 检测到用户取走新卡
- 更新系统卡片信息
- 标记原卡为作废状态
- 跳转 CompletionPage

## 七、实现注意事项

### 7.1 费用计算

```csharp
public class ReplacementFeeCalculator
{
    public decimal CardCost { get; set; } = 20.00m;  // 工本费
    
    public ReplacementFee Calculate(UserInfo user)
    {
        var oldCardBalance = user.Card?.Balance ?? 0;
        var amountDue = Math.Max(0, CardCost - oldCardBalance);
        
        return new ReplacementFee
        {
            CardCost = CardCost,
            OldCardBalance = oldCardBalance,
            AmountDue = amountDue,
            CanDirectDeduct = oldCardBalance >= CardCost
        };
    }
}
```

### 7.2 制卡流程

与 TakeCardPage 类似，但制卡数据不同：
- 新卡号生成
- 余额转移（从旧卡到新卡）
- 原卡状态更新为"已作废"
