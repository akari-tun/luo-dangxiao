## 四、功能描述

### 4.1 页面用途

RechargePage 为教职工提供卡片余额充值功能。支持选择固定金额或输入自定义金额，然后跳转到支付页面完成充值。

### 4.2 核心功能

1. **金额选择**
   - 提供固定金额选项（20, 50, 100, 200, 500）
   - 支持自定义金额输入
   - 金额范围限制（1-5000元）

2. **余额预览**
   - 显示当前余额
   - 实时计算并显示充值后余额

3. **支付跳转**
   - 确认金额后跳转 QRCodePage
   - 展示支付二维码

### 4.3 充值流程

```
显示教职工信息
    ↓
显示当前余额
    ↓
用户选择/输入充值金额
    ↓
显示充值后余额预览
    ↓
用户点击"去支付"
    ↓
跳转 QRCodePage 展示支付二维码
    ↓
支付成功
    ↓
跳转到 CardProcessPage 完成充值
    ↓
跳转到 CompletionPage 完成
```

## 五、交互描述

### 5.1 金额选择

```
用户点击固定金额按钮
    ↓
按钮变为选中状态
    ↓
更新充值后余额预览
    ↓
"去支付"按钮启用
```

### 5.2 自定义金额

```
用户点击"其他"按钮
    ↓
弹出自定义金额输入框
    ↓
用户输入金额
    ↓
验证金额范围（1-5000）
    ↓
有效：更新预览
无效：显示错误提示
    ↓
点击"确定"
    ↓
关闭弹窗，更新金额
```

### 5.3 详细交互

**1. 页面加载**:
- 从 Session 获取教职工信息
- 嵌入 StaffInfoPage 模块
- 显示当前余额
- 默认不选中任何金额
- "去支付"按钮禁用

**2. 选择固定金额**:
- 点击金额按钮
- 取消其他按钮选中状态
- 高亮当前选中按钮
- 计算并显示充值后余额
- 启用"去支付"按钮

**3. 自定义金额**:
- 点击"其他"按钮
- 弹出金额输入对话框
- 输入数字（仅允许数字）
- 实时验证范围
- 点击确定确认

**4. 去支付**:
- 验证金额有效
- 携带金额参数
- 跳转 QRCodePage

**5. 返回操作**:
- 返回 HomePage

## 七、实现注意事项

### 7.1 金额验证

```csharp
public class RechargeValidator
{
    public const decimal MinAmount = 1;
    public const decimal MaxAmount = 5000;
    
    public static bool IsValidAmount(decimal amount)
    {
        return amount >= MinAmount && amount <= MaxAmount;
    }
    
    public static string GetValidationMessage(decimal amount)
    {
        if (amount < MinAmount)
            return $"充值金额不能低于 {MinAmount} 元";
        if (amount > MaxAmount)
            return $"充值金额不能超过 {MaxAmount} 元";
        return null;
    }
}
```

### 7.2 金额选项配置

```csharp
public class RechargeOptions
{
    public List<decimal> FixedAmounts { get; set; } = new List<decimal>
    {
        20, 50, 100, 200, 500
    };
    
    public bool AllowCustomAmount { get; set; } = true;
    public decimal MinAmount { get; set; } = 1;
    public decimal MaxAmount { get; set; } = 5000;
}
```
