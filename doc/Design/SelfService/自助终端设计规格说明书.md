## 二、页面架构与关系

### 2.1 页面分类
- 主页面：HomePage
- 功能子页面：VerifyPage、TakeCardPage、ReportLossPage、ReplacementPage、QueryPage、RechargePage、CheckInPage、CardProcessPage、CompletionPage
- 模块页面：StudentInfoPage、StaffInfoPage、IDCardVerifyPage、SMSVerifyPage

### 2.2 主页面状态
1. 教职工自助状态（StaffSelfService）：显示教职工功能按钮。
2. 学员自助状态（StudentSelfService）：显示学员功能按钮。
3. 子页面容器状态（SubPageContainer）：隐藏功能按钮，展示子页面容器。

### 2.3 页面加载机制
- HomePage 作为主容器，包含顶部栏、内容区、底部区域。
- 功能子页面加载到 HomePage 子页面容器。
- StudentInfoPage/StaffInfoPage 与 IDCardVerifyPage/SMSVerifyPage 作为模块嵌入。

---

## 三、功能流程
### 3.1 教职工自助流程
- 自助取卡：HomePage -> VerifyPage -> TakeCardPage -> CardProcessPage -> CompletionPage
- 自助挂失：HomePage -> VerifyPage -> ReportLossPage -> CardProcessPage -> CompletionPage
- 自助补卡：HomePage -> VerifyPage -> ReplacementPage -> CardProcessPage -> CompletionPage
- 自助查询：HomePage -> VerifyPage -> QueryPage
- 自助充值：HomePage -> VerifyPage -> RechargePage -> CardProcessPage -> CompletionPage

### 3.2 学员自助流程
- 自助取卡：HomePage -> VerifyPage -> TakeCardPage -> CardProcessPage -> CompletionPage
- 自助挂失：HomePage -> VerifyPage -> ReportLossPage -> CardProcessPage -> CompletionPage
- 自助补卡：HomePage -> VerifyPage -> ReplacementPage -> CardProcessPage -> CompletionPage
- 自助报到：HomePage -> VerifyPage -> CheckInPage -> CompletionPage

### 3.3 流程说明
1. 所有功能从 VerifyPage 开始。
2. 验证成功后进入对应功能子页面。
3. 涉及卡片操作时进入 CardProcessPage。
4. 充值流程包含支付二维码与支付确认。
5. 所有流程在 CompletionPage 结束。

## 四、页面设计详述
### 4.1 HomePage
- 三种状态切换：教职工自助 / 学员自助 / 子页面容器。

### 4.2 VerifyPage
- 统一身份验证入口。
- 支持身份证验证与短信验证切换。

### 4.3 功能子页面
- TakeCardPage、ReportLossPage、ReplacementPage、QueryPage、RechargePage、CheckInPage、CardProcessPage、CompletionPage。
- 根据用户类型嵌入 StudentInfoPage 或 StaffInfoPage。

### 4.4 模块页面
- StudentInfoPage：学员信息展示。
- StaffInfoPage：教职工信息展示。
- IDCardVerifyPage：身份证读取与验证。
- SMSVerifyPage：短信验证码验证。
